# UI 草案

本文档给后续 UI 迭代提供可执行的风格方向。当前网页原型已经采用像素字体、城市地图和右侧经营面板，后续可以沿着同一视觉语言继续扩展。

## 视觉方向

关键词：90 年代经营模拟、像素城市、深色工作台、高对比数据、中文默认、英文切换。

| 元素 | 建议 |
| --- | --- |
| 主色 | 深黑蓝背景、绿色经营状态、金色得分、红色风险、青色操作反馈 |
| 字体 | `Press Start 2P` + 等宽 fallback |
| 图形 | CSS 像素格、道路、建筑块、少量像素图标 |
| 信息密度 | 桌面端偏密集，移动端单列展开 |
| 卡片半径 | 像素直角或 2px 小圆角 |
| 反馈 | 按钮按下位移、风险左边条、事件标签 |

## 屏幕 1：创业准备

```text
┌──────────────────────────────────────────────────────────┐
│ 我是老板 / I am boss                    [中文] [English] │
│ 选择创始人画像，设置公司的第一批公司能力。               │
├──────────────────────────────────────────────────────────┤
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐       │
│ │ 技术型创始人 │ │ 人脉型创始人 │ │ 抗压型创始人 │       │
│ │ 技术 +2      │ │ 沟通 +1      │ │ 抗压 +2      │       │
│ │ 智商 +1      │ │ 情商 +1      │ │ 沟通 +1      │       │
│ │ 知名度 +1    │ │ 现金 +30000  │ │ 现金 +15000  │       │
│ └──────────────┘ └──────────────┘ └──────────────┘       │
└──────────────────────────────────────────────────────────┘
```

交互重点：

- 三张大按钮直接开始游戏。
- 卡片展示能力加成和公司加成。
- 默认中文，右上角语言切换。

## 屏幕 2：经营主界面

```text
┌──────────────────────────────────────────────────────────┐
│ 状态：经营中                              阶段：经营中    │
├──────────────────────────────────────────────────────────┤
│ 现金 │ 估值 │ 得分 │ 人数 │ 文化 │ 士气 │ 知名度 │ 周期 │
├───────────────────────────────┬──────────────────────────┤
│                               │ 创始人能力                │
│      像素城市地图             │ 技术/经验/抗压/沟通/EQ/IQ │
│  公司 银行 交易所             ├──────────────────────────┤
│  劳动力市场 法院 政策办公室   │ 人员配比和员工明细        │
│                               ├──────────────────────────┤
│                               │ 文化策略                  │
│                               ├──────────────────────────┤
│                               │ 招聘台：目标工资/报价     │
│                               ├──────────────────────────┤
│                               │ 财务台：跑道/贷款/IPO     │
│                               ├──────────────────────────┤
│                               │ 操作按钮和事件流          │
└───────────────────────────────┴──────────────────────────┘
```

交互重点：

- 城市地图承载公司、银行、交易所、劳动力市场、法院、政策办公室。
- 右侧面板承载高频操作，减少页面跳转。
- 事件流支持类型筛选，显示天数、分类、严重度和文案。
- 员工行显示工资、目标工资、任期、离职风险、加薪、解雇。

## 屏幕 3：游戏结束

```text
┌──────────────────────────────────────────────────────────┐
│                         GAME OVER                        │
│                      原因：破产/退休/死亡                 │
│                                                          │
│                         总分                             │
│                       12,345,678                         │
│                                                          │
│  经营天数 ×1      公司估值 ×2      创始人身价 ×1          │
│                                                          │
│       [再来一局]                    [查看排行榜]          │
└──────────────────────────────────────────────────────────┘
```

交互重点：

- 展示结局原因和 1:2:1 评分拆解。
- 排行榜高亮当前记录。
- 结算页保留像素 scanline 和高对比标题。

## 移动端布局

移动端使用单列顺序：

1. 标题和语言切换
2. 核心 HUD
3. 城市地图
4. 招聘和财务
5. 人员与文化
6. 操作按钮
7. 事件流
